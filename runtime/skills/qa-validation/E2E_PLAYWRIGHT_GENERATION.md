# E2E Playwright Test Generation

## Overview

This extends the QA validation with **persistent test generation** — creating `.spec.ts` files that are committed alongside implementation code and run in CI. This uses a three-agent pattern:

1. **Planner** — generates a Markdown test plan from acceptance criteria
2. **Generator** — creates `.spec.ts` files validated against the live DOM
3. **Healer** — auto-repairs selector drift when tests fail

## When to Generate Tests

Generate persistent E2E tests when:
- The ticket involves user-facing UI changes
- Acceptance criteria describe interactive user workflows
- The enriched ticket has e2e test scenarios
- The project already has a `playwright.config.ts` or uses Playwright

Do NOT generate E2E tests for:
- API-only changes
- Internal utility/library changes
- Configuration changes

## Step 1: Test Plan (Planner)

For each e2e test scenario in the enriched ticket, create a test plan:

```markdown
## E2E Test Plan — <ticket-id>

### Test: <test_name>
- **Scenario:** <description from enriched ticket>
- **Steps:**
  1. Navigate to <URL>
  2. Wait for <element> to be visible
  3. Click <element>
  4. Fill <input> with <value>
  5. Assert <expected outcome>
- **Selectors (preferred order):**
  1. `getByRole('button', { name: 'Submit' })` — accessibility
  2. `getByTestId('submit-btn')` — data-testid
  3. `getByText('Submit')` — text content
- **Assertions:**
  - `expect(page.getByText('Success')).toBeVisible()`
```

### Selector Strategy (Priority Order)
1. **Role-based** — `getByRole()` — most stable, accessibility-friendly
2. **Test ID** — `getByTestId()` — explicit, doesn't break on text changes
3. **Text content** — `getByText()` — readable but fragile if copy changes
4. **Label** — `getByLabel()` — good for form fields
5. **CSS selector** — `page.locator('.class')` — last resort, most fragile

## Step 2: Test Generation (Generator)

Create `.spec.ts` files in the project's e2e test directory:

```typescript
// e2e/<test-name>.spec.ts
import { test, expect } from '@playwright/test';

test.describe('<Feature Name>', () => {
  test('<test scenario description>', async ({ page }) => {
    // Navigate
    await page.goto('/path');

    // Interact
    await page.getByRole('button', { name: 'Toggle' }).click();

    // Assert
    await expect(page.getByText('Dark mode enabled')).toBeVisible();
  });
});
```

### Generated Test Requirements
- Each test must be self-contained (no shared state between tests)
- Use `test.beforeEach` for common setup (navigation, login)
- Use `expect` assertions with `.toBeVisible()`, `.toHaveText()`, `.toHaveAttribute()`
- Add `test.describe` blocks matching the feature name
- Include `// Generated by Agentic Harness — SCRUM-XXX` comment at top

### Validation Against Live DOM
Before committing generated tests:
1. Start the dev server
2. Run each generated test against the live app
3. Fix any selector issues (wrong role, missing testid, changed text)
4. Only commit tests that pass

## Step 3: Healer (Auto-repair)

When a generated test fails due to selector drift:

1. **Detect the failure type:**
   - `locator.click: Target closed` → page navigation changed
   - `Timed out waiting for selector` → element not found
   - `expect: received false` → assertion failed

2. **Get the current DOM state:**
   - Take a snapshot of the page via Playwright MCP
   - Read the accessibility tree
   - Identify the correct current selector

3. **Update the selector:**
   - Replace the broken selector with the correct one
   - Prefer upgrading to a more stable selector type (CSS → role-based)
   - Re-run the test to verify the fix

4. **Commit the healed test:**
   - Include a comment: `// Healed: <old selector> → <new selector>`

## File Placement

```
<project-root>/
  e2e/                          # or tests/e2e/, playwright/
    <feature-name>.spec.ts      # Generated test file
    fixtures/                   # Test data/fixtures (if needed)
  playwright.config.ts          # Playwright configuration
```

Follow the project's existing test directory structure. If no e2e directory exists, create `e2e/` at the project root.

## CI Integration

Generated tests should be runnable in CI:

```yaml
# In GitHub Actions or similar
- name: Run E2E tests
  run: npx playwright test
  env:
    BASE_URL: http://localhost:3000
```

Ensure `playwright.config.ts` has:
- `webServer` config to auto-start the dev server
- Headless browser configuration
- Reasonable timeouts (30s for navigation, 10s for assertions)
