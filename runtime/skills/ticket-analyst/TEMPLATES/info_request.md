# Info Request Template

## Guidelines

When requesting information from the ticket author:

1. **Be specific** — Ask for exactly what you need. "Can you clarify the requirements?" is useless. "What should happen when the user uploads a file larger than the limit — show an error message or silently reject?" is actionable.

2. **Explain why** — Briefly state why this information is needed for implementation: "Without knowing the max file size, we cannot implement the validation logic or write accurate test cases."

3. **Limit questions** — Ask 2-4 targeted questions maximum. If you have more, the ticket is likely too vague and may need to be rewritten.

4. **Don't ask the obvious** — If the answer can be reasonably inferred from the ticket description, existing codebase conventions, or industry standards, infer it instead of asking.

## Format

Questions should be structured as:
- A clear question
- Why it matters for implementation (optional, for non-obvious questions)
